# Patterns

Version: 0.1.0
Last updated: 2026-06-04

Reusable operating patterns for agentic AI work.

Patterns are lighter than frameworks and less prescriptive than skills. Use them when you need a recognizable shape for a workflow, review loop, delegation plan, or safety boundary, but still want to adapt the details to the task.

## Catalog

| Pattern Set | Use It For | Start Here |

| Agent workflow patterns | Choosing how to split, route, verify, or repeat multi-agent work. | [agent-workflow-patterns](agent-workflow-patterns/README.md) |

## Public Boundary

Patterns in this directory should stay portable. Avoid private project names, customer examples, local paths, raw session exports, secrets, or assumptions about a specific agent runtime unless the pattern explicitly depends on that runtime.
